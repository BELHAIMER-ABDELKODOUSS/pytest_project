Installation

To install Pytest, you need to have Python installed on your system. If you don't have Python installed, you can download and install it from the official Python website (https://www.python.org/). :

Once you have Python installed, you can use pip, the package manager for Python, to install Pytest. To install Pytest using pip, follow these steps:

  1. Open a terminal or command prompt window
  2. Make sure that you have the latest version of pip by running the following command: pip install --upgrade pip
  3. Install Pytest by running the following command: pip install pytest

  4. That's it! Pytest should now be installed on your system and ready to use. You can verify that Pytest is installed by running the pytest --version command, which should print the version number of Pytest that you have installed. Alternatively, you can also install Pytest from source by downloading the source code from the Pytest GitHub repository (https://github.com/pytest-dev/pytest) and running the python setup.py install command. However, this method is not recommended unless you have a specific reason for installing Pytest from source.

Create your first test

To create a test with Pytest, you need to do the following:

  1. Create a Python file in your project directory. This file will contain your test functions.
  2. Import the pytest module.
  3. Define a test function by decorating it with the @pytest.mark.test decorator.
  4. Write the code for your test inside the test function.
  5. Run the test using the pytest command.

Here's an example of a simple test function that checks if the result of the add function is correct: 


# test_example.py;
import pytest

def add(x, y):
    return x + y

def test_add():
    assert add(2, 3) == 5

To run this test, you can use the pytest command from the command line: 


$ pytest test_example.py

This will run the test_add test function and print the result. If the test passes, you should see a message like this: 

== 1 passed in 0.01s ==

If the test fails, you will see an error message indicating what went wrong. That's it! You've just written and run your first test with Pytest. You can now add more test functions to your test file and run them using the pytest command.

Run multiple tests

To run multiple tests using Pytest, you can either:

  1. Create multiple test functions in the same Python file.
  2. Create multiple test files and run them all at once.

Here's an example of how you can create multiple test functions in the same Python file: 


# test_example.py
import pytest

def add(x, y):
    return x + y

def test_add():
    assert add(2, 3) == 5

def test_add_negative():
    assert add(-1, -1) == -2

To run all of these tests, you can use the pytest command from the command line: 


$ pytest test_example.py

This will run both the test_add and test_add_negative test functions and print the result. Alternatively, you can also create multiple test files and run them all at once. For example, you might have a file called test_add.py that contains tests for the add function, and another file called test_subtract.py that contains tests for the subtract function. To run all of these tests, you can use the pytest command followed by a directory path: 

$ pytest tests/

This will run all tests in the tests directory and its subdirectories. You can also use the -k option to specify a test name pattern to filter the tests that you want to run. For example, to run only the tests that contain the word "add" in their names, you can use the following command: 


$ pytest -k add

This will run all tests that match the pattern "add" in their names, regardless of which file they are in.

Assert that a certain exception is raised

To assert that a certain exception is raised in Pytest, you can use the pytest.raises function. This function allows you to test whether a block of code raises an expected exception. Here's an example of how you can use pytest.raises to test for an exception: 

             
import pytest

def test_division_by_zero():
    with pytest.raises(ZeroDivisionError):
        1 / 0

In this example, the test function uses the pytest.raises function to wrap a block of code that divides 1 by 0. This block of code is expected to raise a ZeroDivisionError, and the pytest.raises function will catch this exception and consider the test to be a success. If the exception is not raised, or if a different exception is raised, the test will fail. You can also use the pytest.raises function to test for specific exception messages or to specify the expected exception type as an argument. For example: 


import pytest

def test_division_by_zero():
    with pytest.raises(ZeroDivisionError, match='division by zero'):
        1 / 0

This test will only pass if a ZeroDivisionError is raised with the message "division by zero". You can also test for multiple exception types by using a tuple of exception types as the argument to pytest.raises. For example: 


import pytest

def test_division_by_zero():
    with pytest.raises((ZeroDivisionError, TypeError)):
        1 / 'a'

This test will pass if either a ZeroDivisionError or a TypeError is raised.

Setup & Testing

To set up pytest in a Django project, you will need to do the following steps: :

  1. Install the pytest and pytest-django packages using pip: 


    2. 
pip install pytest pytest-django
  2. Create a tests directory at the root level of your Django project. This is where you will put your tests.
  3. Point pytest to your Django settings

    4. You need to tell pytest which Django settings should be used for test runs. The easiest way to achieve this is to create a pytest configuration file with this information.

  4. Create a file called pytest.ini in your project root directory that contains: 
    5. 
[pytest]
DJANGO_SETTINGS_MODULE = yourproject.settings

python_files = tests.py test_*.py *_tests.py
